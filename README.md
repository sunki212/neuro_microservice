# neuro_microservice

# Описание проекта
Этот проект представляет собой распределенную систему для обработки машинного обучения, построенную на основе микросервисной архитектуры с использованием RabbitMQ в качестве брокера сообщений. Система генерирует данные, выполняет предсказания с помощью ML-модели, вычисляет метрики качества и визуализирует результаты.

# Архитектура
Система состоит из следующих сервисов:

1. RabbitMQ - брокер сообщений для асинхронной коммуникации между сервисами

2. Features Service (features.py) - генерирует данные и отправляет их в очередь

3. Model Service (model.py) - получает данные, делает предсказания с помощью ML-модели

4. Metric Service (metric.py) - вычисляет метрики ошибок и логирует их

5. Plot Service (plot.py) - создает визуализации на основе собранных метрик

# Структура проекта

├── docker-compose.yml            #Конфигурация Docker Compose для всех сервисов  
├── requirements.txt            #Зависимости Python  
├── features.py                 #Сервис генерации данных  
├── model.py                    #Сервис ML-модели  
├── metric.py                   #Сервис вычисления метрик  
├── plot.py                     #Сервис визуализации  
├── logs/                       #Директория для логов и графиков  
│   ├── metric_log.csv          #CSV файл с метриками  
│   └── error_distribution.png  #График распределения ошибок  
└── Dockerfile                  #Dockerfile для сборки сервисов

# Установка и запуск
1. Клонируйте репозиторий:  
``` bash
git clone https://github.com/sunki212/neuro_microservice
```  
```
cd neuro_microservice
```
     
2. Запустите все сервисы:
```
docker-compose up --build
``` 
# Результат работы
После запуска всех сервисов в папке ./logs на машине появятся два файла:  
1. metric_log.csv - таблица метрик   
2. error_distribution.png - график распределения ошибок, обновляемый в реальном времени
